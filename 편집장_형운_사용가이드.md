# 편집장 형운 사용 가이드

## 빠른 시작

### 기획안 피드백 받기

**Step 1:** 기획안을 Claude Code에 복붙합니다

```
제목: [제목]
부제: [부제]

기획의도: [기획의도]

고민 포인트: [있으면 작성 (선택)]

서문:
[서문 초안]

목차:
1. [섹션 1]
2. [섹션 2]
3. [섹션 3]
```

**Step 2:** 아래와 같이 요청합니다

```
>> 형운님, 이 기획 어떤가요?
```

**받게 될 피드백:**
- 팩트 체크 (검증된 정보만 사용) - **필수**
- 따뜻한 인사와 안부 인사 (스몰토크) - 에디터 이름이 있을 때만
- 당신의 고민점에 대한 직접적인 답변
- 핵심 문제점 지적 (냉철하되 예의 있게)
- **독자 관점**: 이 기획이 독자에게 어떤 가치를 제공하는가?
- **동료 관점**: 다른 에디터들이 이 기획안을 명확히 이해할 수 있는가?
- 잘된 부분 인정 (간결하게)
- 성장 기회로서의 개선 과제 제시
- 다음 스텝에 대한 구체적인 액션

---

### 최종 아티클 피드백 받기

**Step 1:** 발행된 아티클 URL을 준비합니다

**Step 2:** 아래와 같이 요청합니다

```
>> 형운님, 이 아티클 피드백 부탁합니다
https://square.antiegg.kr/article-url
```

**받게 될 피드백:**
- 따뜻한 인사와 안부 인사 (스몰토크)
- 콘텐츠 품질 평가 (냉철하게)
- SEO 실질 이슈 지적
- ANTIEGG 스타일 부합도 검토
- 강점 인정 (간결하게)
- 우선순위별 개선 제안

## 기획안 평가 항목

형운이 확인하는 7가지:

### 1️⃣ 주제 타당성
- ANTIEGG 독자가 관심 가질 만한 주제인가?
- 문화예술 분야와 연결되어 있는가?
- 독자에게 줄 가치가 있는가?

### 2️⃣ 방향성 & 차별화
- 기획의도가 명확한가?
- 기존과 다른 관점을 담고 있는가?
- 독특한 접근이 있는가?

### 3️⃣ 구성안의 논리성
- 제목과 부제가 명확하고 직관적인가? (문학성보다 명확함이 우선)
- 목차의 흐름이 자연스러운가?
- 각 섹션이 논리적으로 연결되는가?

### 4️⃣ SEO 잠재력
- 검색하고 싶어질 만한 제목인가?
- 주요 키워드가 자연스럽게 들어가 있는가?
- 독자가 한눈에 "이 글이 뭐에 관한 것인지" 알 수 있는가?

### 5️⃣ ANTIEGG 스타일 부합도
- ANTIEGG의 톤과 맞는가?
- 우리 독자들이 공감할 관점인가?
- 매거진의 정체성을 해치지 않는가?

### 6️⃣ 독자 관점 - 가치와 의미
- 독자에게 어떤 "발견" 또는 가치를 제공하는가?
- 왜 "지금" 이 주제가 필요한가?
- 기획안을 읽는 입장에서 "왜 이 기획이 중요한지" 명확한가?

### 7️⃣ 동료 이해도 - 기획안의 명확성
- 다른 에디터가 이 기획안을 봤을 때 이해하기 쉬운가?
- 각 섹션 간의 연결고리가 명확한가?
- 필요한 사례나 배경 정보가 충분히 제시되어 있는가?

## 최종 아티클 평가 항목

### 콘텐츠 품질
- 주요 메시지가 명확하게 전달되는가?
- 논리적 흐름이 있는가?
- 정보의 깊이와 신뢰성이 있는가?
- 읽을 가치가 있는 콘텐츠인가?

### SEO 기술 요소
- **제목 태그**: 30-60자 범위, 키워드 포함, 클릭 유도
- **메타 디스크립션**: 150-160자, 핵심 요약
- **헤딩 구조**: H1 1개, H2/H3 논리적 배치
- **키워드 최적화**: 자연스러운 배치
- **가독성**: 문단 길이, 문장 길이, 모바일 레이아웃
- **Open Graph 태그**: 소셜 공유 최적화
- **Schema 마크업**: 구조화 데이터

### ANTIEGG 스타일 부합도
- 톤과 스타일이 일관성 있는가?
- 매거진 브랜드와 맞는가?
- 독자들이 기대하는 감성이 있는가?

## 자주 묻는 질문

### Q. 형운이 언제 피드백을 줍니까?
**A.** 기획안이나 URL을 보내면 즉시 피드백을 드립니다. (실시간)

### Q. 기획안을 몇 번이고 물어도 됩니까?
**A.** 네, 물론입니다! 다양한 각도에서 계속 피드백을 드릴 수 있습니다.

### Q. SEO는 뭐가 중요한가요?
**A.**
- **제목 태그** (30-60자): 검색 결과에 보이는 제목
- **메타 디스크립션** (150-160자): 검색 결과 아래 설명
- **헤딩 구조**: 독자와 검색 엔진이 이해하는 구조
- **키워드**: 자연스럽게 분포되어 있는가

### Q. ANTIEGG 스타일이 뭡니까?
**A.**
- 큐레이션된 감각 (최고만 선택)
- 세련되고 미니멀한 표현
- 친근하면서 전문적인 톤
- 독자의 발견 욕구를 자극

### Q. 고민점(고민 포인트)을 안 쓰면요?
**A.** 괜찮습니다! 형운이 전반적으로 평가해드립니다.
다만 특정 부분에 대한 조언이 필요하면 쓰는 게 좋습니다.

### Q. 아티클이 아직 발행 안 됐는데 피드백받을 수 있나요?
**A.** 네, 기획안 형태로 텍스트를 보내면 됩니다!

## 예시 보기

### 기획안 피드백 예시

**당신의 입력:**
```
제목: 무한의 시대에 유한성으로 답하는 법
부제: 스트리밍 시대에도 물성을 고수하는 이들은 '러다이트'일까?

기획의도:
기술 발전의 반대쪽에 서 있는 사람들을 '구시대적'이 아니라
'능동적 선택'으로 봅니다.

고민 포인트:
3번 목차를 뒷받침할 사례가 부족한데,
좋은 사례가 있을까요?

목차:
1. 러다이트 개념 설명
2. 스트리밍 vs 물성의 철학
3. 러다이트는 시대 뒤처진 사람들인가?
```

**형운의 응답:**
```
읽어보니 기획의도는 명확한데, 구성 논리가 좀 더 강해져야 할 것 같아요.

다만 제목은 정말 좋습니다.
"구시대적 고집"이 아니라 "능동적 선택"으로 프레이밍하신 부분도 센스있고요.

고민되는 부분:
- 1번, 2번, 3번이 어떤 인과관계로 이어지는지가 명확하지 않습니다
- 현재는 "좋은 사례 3개 + 결론" 구조처럼 느껴져요
- "러다이트는 시대 뒤처진 사람들인가?"라는 물음에 세 사례가 어떻게 답하는지 더 분명해야 할 것 같습니다

제안:
1. 먼저 기획안 단계에서 각 섹션 간 연결고리를 명확히 하세요
2. "러다이트 = 능동적 선택"이라는 주장을 증명하기 위해 각 사례가 뭘 보여주는지 정리하세요
3. 그 다음 아티클 작성을 시작하면 논리가 훨씬 견고해질 거예요

SEO는 제목 측면에서 잘 되어 있습니다.
화이팅!
```
